# ADR-0005 — Traceability matrix and session journaling

## Context

The repository must answer, at any time, for any atomic unit:

- Which REQs cover this Epic?
- Which tests cover this REQ?
- Which commits added the spec / added the failing test / made it pass / made
  it deprecated?
- Which task was active when the last edit was made?
- If a session crashed mid-stream, where does the next session resume?

The naive solution — a hand-maintained matrix file — fails the moment it gets
out of sync. The reliable solution is to make the matrix a **derived cache**
over append-only sources of truth.

## Decision

### 1. Sources of truth (ordered)

1. **Git history** — commit objects, including mandatory message trailers
   (below).
2. **Spec files** — REQ blocks with YAML frontmatter as defined in
   [ADR-0004](ADR-0004-spec-storage-discipline.md).
3. **Test annotations** — every test declares `@covers REQ-X-NNNN [, ...]` in
   a docstring or decorator; `@test-id TEST-X-NNNN`.
4. **Session journals** — `.claude/journal/session-<id>.jsonl`, append-only,
   one event per line.

### 2. The matrix is a cache

`openspec/traceability/matrix.yaml` is regenerated by
`tools/trace/rebuild.py` from the four sources above. The file is checked in
so that diffs are reviewable, but:

- No human or agent may hand-edit it.
- A pre-commit hook re-runs rebuild and stages the result.
- CI re-runs rebuild on PR and fails if the committed matrix differs.

### 3. Mandatory commit message trailers

Every commit (except checkpoint commits, below) must carry:

```
Task: TASK-<NNNN>
Requirements: REQ-<X>-<NNNN>[, REQ-<Y>-<NNNN>, ...]
Tests-Status: <none|red|red→green|green|deprecated>
Phase: PHASE-<N>
```

The `PreToolUse(Bash:git commit)` hook validates trailers and rejects commits
that violate the rules in §4.

### 4. Status transition rules (P4 enforcement)

- A test's first green commit must be preceded by at least one commit where
  the same test exists in a red state. `tools/trace/validate_commit.py`
  walks git history to verify.
- A REQ may not skip from `draft` to `tests-green`; it must pass through
  `tests-red`.
- A REQ marked `deprecated` may not be re-activated; supersession creates a
  new REQ.

### 5. Journal events

Append-only JSONL. Each line is a complete, parseable event; partial writes
from a crash are discarded by the parser (the line is malformed and ignored).

| Event              | Fields                                                                                  |
|--------------------|-----------------------------------------------------------------------------------------|
| `session_start`    | `ts, session_id, head_sha, resumes_task?`                                               |
| `task_activate`    | `ts, task_id, req_ids[]`                                                                |
| `task_pause`       | `ts, task_id, reason`                                                                   |
| `task_complete`    | `ts, task_id`                                                                           |
| `file_touch`       | `ts, path, tool, task_id?`                                                              |
| `tests_run`        | `ts, runner, passed, failed, new_red[], new_green[]`                                    |
| `commit`           | `ts, sha, task_id, req_ids[], trailers_valid, tests_status`                             |
| `push`             | `ts, branch, ref, success`                                                              |
| `session_end`      | `ts, clean, active_task?`                                                               |

### 6. Hooks (configured in `.claude/settings.json`)

| Event                          | Hook                                  | Purpose                                                  |
|--------------------------------|---------------------------------------|----------------------------------------------------------|
| `SessionStart`                 | `tools/trace/resume.py`               | Print active phase, in-flight task, last commit, next action |
| `PreToolUse(Bash:git commit)`  | `tools/trace/validate_commit.py`      | Reject invalid commit messages; enforce P4               |
| `PostToolUse(Bash:git commit)` | `tools/trace/journal_commit.py`       | Append `commit` event                                    |
| `PostToolUse(Edit\|Write)`     | `tools/trace/journal_touch.py`        | Append `file_touch` event                                |
| `PostToolUse(Bash:*test*)`     | `tools/trace/journal_tests.py`        | Parse test results; append `tests_run` event             |
| `Stop` / `SessionEnd`          | `tools/trace/checkpoint.py`           | If working tree dirty: WIP commit + push; append `session_end` |

Hook implementations land in PHASE-2. PHASE-0 commits the `.claude/settings.json`
declarations as placeholders (commands that exit 0 with a TODO comment).

### 7. Checkpoint commits

The `Stop` / `SessionEnd` hook produces:

```
wip: checkpoint <session-id>

Task: TASK-<active>
Requirements: <REQ ids of active task>
Tests-Status: none
Phase: PHASE-<N>
Checkpoint: true
```

The `Checkpoint: true` trailer makes CI gates skip the commit and makes the
next non-checkpoint commit on the branch responsible for restoring P4
compliance (either by squash or by ensuring the red/green sequence is
recoverable from the full history).

### 8. Failure modes and worst case

| Failure                                    | Loss                                                              |
|--------------------------------------------|-------------------------------------------------------------------|
| Crash mid-Edit/Write, container survives   | Zero on disk (Edit writes atomically by rename).                  |
| Crash mid-commit                           | Zero (git commit is atomic; rebuild fills missing journal entry). |
| Container reclaim, push succeeded          | Zero.                                                             |
| Container reclaim, uncommitted             | Mitigated by checkpoint hook; otherwise WIP files lost.           |
| Crash mid-tool-call before checkpoint hook | At most the in-flight tool call's effect.                         |

Claude's reasoning context between tool calls is not persisted. Recovery is by
reconstruction from journal + active task file + last commit; a fresh session
needs zero human re-briefing.

### 9. Audit interface

`tools/trace/audit.py <id>` answers any of the following deterministically:

- `audit REQ-X-NNNN` → spec file, tests, tasks, commits, phase, status.
- `audit TASK-NNNN` → owning change, REQs, commits, status, owning session.
- `audit STORY-NNNN` → REQs, tests, completion %.
- `audit EPIC-NNNN` → coverage %, gate status.
- `audit PHASE-N` → exit criteria status.

## Consequences

- Matrix can be regenerated from sources at any time; corruption is
  recoverable.
- Hand-editing the matrix is a build failure, not a merge conflict — the
  failure mode is loud, not silent.
- Commit-message discipline becomes infrastructural; the cost is one trailer
  block per commit, the benefit is durable traceability.
- Session resume is mechanical; no human briefing required.

## Compliance

- `REQ-ARCH-0008` confirms hooks are declared from PHASE-0.
- PHASE-2 delivers `tools/trace/*` implementations with red-first tests.
- CI gate `matrix-drift` (PHASE-2) blocks PRs whose committed matrix differs
  from rebuild.
