# TASK-0005 — Implement parser + req-id-format + spec-frontmatter-valid

## Intent

Replace the `NotImplementedError` stubs from TASK-0004 with working
implementations. All `tools/spec_lint/tests/*.py` tests turn green.

## Definition of done

- [ ] `parser.parse_spec_file(path)` returns a `SpecFile`:
  - Identifies every `## REQ-…` heading (permissive regex; strict validation
    is rules' job).
  - For each, extracts the immediately-following `--- ... ---` YAML
    frontmatter (or sets `frontmatter = None`).
  - Records `heading_line` as 1-based.
  - Records the file path as both `SpecFile.path` and each `ReqBlock.file`.
- [ ] `rules.req_id_format.ReqIdFormat.check(spec)` returns an empty list when
      every REQ matches `^REQ-[A-Z0-9]{1,12}-\d{4}$`; returns one finding per
      violating heading id and per violating frontmatter id.
- [ ] `rules.spec_frontmatter_valid.SpecFrontmatterValid.check(spec)` enforces:
      every required key (`id`, `revision`, `status`, `introduced`,
      `supersedes`, `phase`, `tier`, `references`) is present; `id` equals
      heading; `status` ∈ enum; `tier` ∈ enum; `revision` is positive int;
      `introduced` matches `CHG-\d{4}`; `phase` matches `PHASE-\d+`;
      `supersedes` is null or REQ-ID; `references` is a mapping with `epic`,
      `story`, `adrs` keys; `references.adrs` is a list.
- [ ] All findings carry `rule_id`, `severity == "error"`, `file`, `req_id`,
      `line`, and `message` populated.
- [ ] `pytest tools/spec_lint/ tools/ci/` reports all tests passing.
- [ ] Running each rule manually against every fixture file matches the
      directory it lives in (valid → 0 findings; invalid → ≥ 1 finding).
