# TASK-0007 — Implement git_history adapter + 2 history-aware rules

## Intent

Replace the NIE stubs from TASK-0006 with working implementations. Tests
turn green.

## Definition of done

- [ ] `git_history.historical_versions(repo, file_rel)`:
  - Lists commits that touched `file_rel` via `git log --format=%H%x00%cI --diff-filter=AM -- <file>`.
  - Reverses to oldest→newest order.
  - For each commit, retrieves content via `git show <sha>:<file>`.
  - Parses content with `parser.parse_spec_file_text` (a sibling that takes
    raw text rather than a path).
  - Yields `HistoricalVersion(sha, timestamp, content, spec_file)`.
- [ ] `rules.req_id_immutable.ReqIdImmutable.check_history(versions)`:
  - Builds, for each version, the set of REQ-IDs present.
  - For each adjacent pair (prev, curr), flags every ID that was in `prev`
    but not in `curr` with a finding citing both SHAs.
- [ ] `rules.req_append_only.ReqAppendOnly.check_history(versions)`:
  - Tracks per-REQ first version where `status == "tests-green"`.
  - For each subsequent version, compares Description and Acceptance bodies
    against the snapshot at first-green.
  - If the body changed and `revision` did not bump beyond the first-green
    revision, flags a finding citing the offending SHA.
- [ ] All findings carry stable `rule_id`, severity `"error"`, file path,
      req_id, line populated (line from the version's parsed SpecFile),
      and a message that names the offending SHA.
- [ ] `pytest tools/spec_lint/ tools/ci/` is all-green.
