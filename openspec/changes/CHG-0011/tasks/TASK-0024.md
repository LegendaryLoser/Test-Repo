# TASK-0024 — eliminate `prose-xref-banned` allow markers via code-fenced examples

## Intent

The TASK-0023 GREEN commit added 5 `<!-- spec-lint: allow prose-xref-banned -->`
markers to lines that quote the rule's forbidden patterns for
pedagogical purposes (ARCHITECTURE.md / ADR-0004 / CHG-0005 proposal +
TASK-0008). Per the chat exchange, these markers are a maintenance
liability: a future writer who copy-pastes a marker-bearing line into a
new context silently suppresses a real check.

Eliminate the markers entirely by relying on `prose_xref_banned`'s
existing fence-block exemption (`_CODE_FENCE` regex skips
triple-backtick blocks). Restructure each documentary line so the
example patterns live inside a fenced code block instead of inline
prose. The rule no longer sees them; no marker needed.
