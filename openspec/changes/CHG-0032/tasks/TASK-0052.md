# TASK-0052 — Wave 8 path-dependence test (Option C): 3 streams + path-dependence-results.md

## Intent

Per user direction from `cell-occupancy.md` §7, executed Option C: path-dependence remediation. Spawned 3 streams adding a SECOND method to each of 3 cells currently occupied by only one Wave-6/7 method, to empirically test whether the first method's Pareto front was complete (saturation hypothesis) or whether counterfactual methods surface clusters the first method missed (path-dependence hypothesis).

`methodology-research-note.md` §2.2 listed path-dependence as a known limitation of the post-hoc-cells framework and listed N-restart bootstrap as the rigorous-but-cost-prohibitive mitigation, with documentation-only as the pragmatic compromise. This experiment is a partial N=2 bootstrap testing whether the cheaper mitigation is empirically defensible.

**Result: path-dependence is substantial — ~30 NEW clusters across 3 streams; pre-W8 saturation estimate (93%) was decisively wrong; revised saturation is 58%. The documentation-only mitigation is insufficient.**

## Methodology implications (per wave-8-path-dependence-results.md §4)

1. **Path-dependence is empirically substantial.** Adding ONE counterfactual method per cell expanded clusters by ~71%. Documentation-only mitigation under-represents the true defect surface by ~40-70%.
2. **Cell map is incomplete by construction.** Wave 8 opened ~9 new cells beyond the 15-cell pre-W8 map. True meaningful_cells denominator is probably 30-40 cells for this audit.
3. **N-restart bootstrap is empirically necessary.** A partial N=2 bootstrap (this experiment) materially changed convergence assessment. The methodology codification ADR should specify minimum-N-second-method-runs-per-occupied-cell as a precondition for any "saturated" claim. <!-- spec-lint: allow prose-xref-banned -->
4. **Saturation requires per-cell Pareto-stability evidence.** A cell isn't saturated just because no new method admitted in K=2 waves; it's saturated when ≥2 methods have produced overlapping (not new-cluster-opening) outputs.

## Updated stopping options

- **A' — Continue path-dependence remediation in Wave-8 cells.** Spawn Wave 9 with second-methods in the 9 new Wave-8 cells. Expected ~15-30 new clusters; saturation might rise to 65-75% but unlikely to reach 95%.
- **B' — Accept exhaustively-explored-but-path-dependent.** Document the corpus as ~72 clusters + path-dependence-fundamental. Pivot to resolution + methodology codification + implementation audit.
- **C' — Bootstrap-style restart.** Re-run Wave 1 with different method-mix and compare cluster overlap. ~2× audit cost.

Recommendation (per `wave-8-path-dependence-results.md` §5): **B' — accept incompleteness, pivot.** Marginal value of each new cluster decreases; continuing waves indefinitely has diminishing ROI. The audit has likely already passed the ROI peak.

## Notes

- **The pre-W8 saturation estimate was a measurement-discipline failure, not a methodology-framework failure.** Saturation per `methodology-research-note.md` §3.1 requires "Pareto front of (σ, κ) stable in last K consecutive waves." Pre-W8 we measured "no new method admitted in K waves" as a proxy for stability, which is wrong: a cell whose only method is the original first-admission has by definition not had its Pareto front tested. Methodology ADR must tighten this.
- **The cell-occupancy.md §6 path-dependence acknowledgment WAS correct but UNDER-CONFIDENT.** It said "the 42-cluster + 15-cell map is the cell map that emerged from THIS method-sequence" but didn't quantify the gap. Wave 8 quantified: ~71% cluster expansion under partial bootstrap.

## Risk

- **NEW-vs-REINFORCEMENT classification of Wave-8 findings is judgment.** §1's ~30/~23 split has ±~5 uncertainty either way. Methodology ADR's audit-meta-test suite should mechanise this via greedy ablation.
- **The 9 "new cells" Wave 8 opened are arguably sub-cells of existing cells.** Methodology ADR's cell-axis discipline (probably the Layer 2 specification for architecture audits) needs to specify when a new (A, B, C, D) tuple counts as a new cell vs a finer-grained labeling of an existing one.
- **The implication "B' — accept incompleteness, pivot" is a normative recommendation, not an empirical conclusion.** Reasonable people could read the data and conclude A' or C' instead. The methodology codification ADR's stopping rule has to land normatively somewhere; this audit's data supports several stopping points.
- **The Wave-8 streams used opus and 1-shot prompts.** Sonnet permutations, different prompt phrasings, or chain-of-thought variations would likely produce yet-different cluster sets. The 72-cluster number is a snapshot of THIS configuration on THIS day.
