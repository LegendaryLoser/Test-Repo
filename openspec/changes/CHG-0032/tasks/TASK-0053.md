## §6 Acceptance criteria (what makes this POC "successful")

The POC validates the framework if:

1. **All 72 clusters fit into v1.0 axes.** ≥95% of clusters cleanly assignable to one axis; ≤5% trigger axis-add candidate (and the candidate is documented but not necessarily added in v1.0).
2. **Trunk closure argument is defensible.** Reviewer can read the "these N trunk cells suffice because..." argument and either accept or flag a specific cell as missing.
3. **Retrospective verdict is honest.** If under v1.0 the 8 waves did NOT fill the trunk, the POC says so explicitly + proposes what additional pass would close the gap.
4. **No unbounded expansion.** v1.0 produces a finite cell space (≤100 admissible cells). If the Cartesian product yields >100, the admissibility filter is tightened until ≤100.
5. **Archetype prior is justified.** Per-axis weight assignment cites either empirical severity data or domain reasoning.

If any of these fail, the POC report explains why + recommends a v1.0 revision OR identifies a framework defect to escalate.
