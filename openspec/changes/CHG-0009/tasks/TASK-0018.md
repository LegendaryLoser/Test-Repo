# TASK-0018 — Implement loaders + runner + report; commit baseline

## Intent

Replace the NIE stubs from TASK-0017 with working implementations. Run the
dry-run; commit the generated `baseline.md` so future runs can diff.

## Definition of done

- [ ] `loaders.load_openspec_docs(corpus_dir)` returns
      `list[tuple[Path, str]]` for every `*.md` in the corpus.
- [ ] `loaders.load_promise_arff(arff_path)` parses the ARFF body,
      extracts each requirement's text and label, returns a list of
      synthetic `SpecFile` objects (one per requirement; text in
      Description; label embedded in a deterministic synthetic ID).
- [ ] `runner.run_dry_run` iterates applicable rules per corpus:
  - OpenSpec docs → `prose-xref-banned`, `xref-resolves`.
  - PROMISE NFR → `anti-aliasing`, `compound-requirement-detector`.
  - Collects `findings` and a separate `uncertain_zone` log keyed by
    rule (anti-aliasing: pairs with sim ∈ [0.5, 0.7); compound: REQs
    with single Given but Description containing "and"; prose: lines
    matching pattern but with same-line ID — already-skipped cases also
    logged for visibility).
- [ ] `report.render_report` produces a markdown report with:
  - Run metadata (rules commit, corpus commits, runtime).
  - Per-corpus section with finding counts per rule.
  - Top-N findings per rule with file:line and message.
  - Uncertain-zone summary per rule.
- [ ] `tools/spec_lint/benchmark/reports/baseline.md` committed; contains
      the report from the current run.
- [ ] All `tools/` tests green.
- [ ] Total benchmark runtime under 30 seconds.
