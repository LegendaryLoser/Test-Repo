# TASK-0011 — Implement compound-requirement-detector + anti-aliasing

## Intent

Replace the NIE stubs from TASK-0010 with working implementations. All tests
turn green.

## Definition of done

- [ ] `CompoundRequirementDetector.check(spec_file)`:
  - For each REQ, extracts Acceptance via `_sections.extract_section`.
  - Counts lines matching ``\bgiven\b`` (case-insensitive, at start of bullet
    or line after stripping leading dash/whitespace).
  - > 1 → one finding per REQ, message states the observed count and the
    ADR-0004 §3 remedy ("split into N REQs").
- [ ] `AntiAliasing.check_corpus(spec_files)`:
  - Construction reads `tools/spec_lint/config.yaml`; constructor also
    accepts an override dict for tests.
  - Builds list of (req_id, file, line, body_text) tuples across all
    SpecFiles where body_text is Description + "\n" + Acceptance.
  - Iterates unordered pairs (i, j with i < j); computes Jaccard over
    character n-grams (lowercase, contiguous, size from config).
  - If similarity ≥ threshold → one finding attributed to the first REQ
    in the pair, naming the partner REQ, partner file:line, and the score.
- [ ] All findings carry rule_id, severity "error", file, req_id, line, and
      a descriptive message.
- [ ] `pytest tools/spec_lint/ tools/ci/` is all-green.
