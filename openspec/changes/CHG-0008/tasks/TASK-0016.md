# TASK-0016 — Fix parser regex; remove invalid status mutation

## Intent

Two defects surfaced by TASK-0015 driver run:

1. **Parser regex too restrictive (rule defect).** `^## (REQ-\S+)\s*$`
   silently drops headings whose ID contains an embedded special character
   (e.g., space, control char). The format rule never sees them. Fix:
   relax to `^## (REQ-[^\n]+?)\s*$` so the parser surfaces any plausibly
   REQ-shaped heading and the format rule judges strictness.

2. **`FM-STATUS-DRAFT` mutation is invalid by design (corpus defect).**
   YAML normalizes trailing whitespace on scalar values, so
   `status: draft ` parses to `"draft"` — a legal enum value. The
   mutation cannot meaningfully test rule behavior. Remove.
